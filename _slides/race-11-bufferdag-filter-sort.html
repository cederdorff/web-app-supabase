Filter & sortering

Hvorfor er det relevant?

Når jeres app har data, skal brugeren kunne finde det rigtige indhold hurtigt.

Filtrering og sortering er ofte forskellen på "data på en side" og en app, der faktisk kan bruges.
Supabase REST

Filter sker med query parameters

Supabase bruger query parameters til at filtrere direkte i databasen.


/rest/v1/products?id=eq.1

/rest/v1/products?price=lt.1000

/rest/v1/products?title=ilike.*mac*

Pointen: Vi henter kun de data, vi faktisk skal bruge.

Operatorer

De mest brugbare i jeres projekt

eq

Lig med. God til detail views og relationer.

lt / gt

Mindre end og større end. God til pris, alder, rating, datoer.

ilike

Tekstsøgning uden forskel på store og små bogstaver.

Sortering

Sortering sker med order


/rest/v1/products?order=price.asc

/rest/v1/products?order=price.desc

/rest/v1/products?order=created_at.desc

Billigste først, dyreste først, nyeste først - samme mønster.

Kombination

Filter, sortering og limit kan kombineres


/rest/v1/products?price=lt.5000&order=price.asc&limit=5

Læs URL'en som en sætning: produkter under 5000, sorteret billigste først, maks 5 resultater.

JavaScript

Det er stadig bare et fetch


const response = await fetch(
  "https://xyz.supabase.co/rest/v1/products?price=lt.5000&order=price.asc&limit=5",
  {
    headers: {
      apikey: "din_sb_publishable_xyz"
    }
  }
);

const data = await response.json();
React

UI styrer query'en

I React kommer filter og sortering typisk fra state: inputfelter, dropdowns, knapper eller tabs.


const price = 5000;
const order = "price";
const direction = "asc";

const url =
  `${baseUrl}/products?price=lt.${price}&order=${order}.${direction}`;

  • Loading og fejlbesked, så brugeren kan mærke hvad der sker.
