# RACE 9 - Web App: HTTP & REST API - 17-04-2026

## Formål

Vi skal forstå grundlæggende HTTP og REST i praksis og arbejde med GET, POST, PATCH og DELETE mod Supabase. Målet er, at vi ser sammenhængen mellem handlinger i databasen og HTTP-requests, før vi arbejder videre med det i React.

## Forberedelse

- Du skal have Thunder Client installeret i VS Code.
- Du skal have dit Supabase-projekt klar med data i `products`-tabellen.
- Du skal læse README-opgaven frem til og med delen om Thunder Client.
- Du skal genopfriske begreberne request, response, headers og body.
- Hav din API key klar.

## Agenda

1. Vi repeterer kort sidste undervisningsgang.
2. Vi introducerer HTTP: request, response, headers, body og statuskoder.
3. Vi introducerer REST og ideen om ressourcer og endpoints.
4. Vi gennemgår Supabase endpointet for `products`.
5. Vi tester GET i Thunder Client.
6. Vi tester POST for at oprette et produkt.
7. Vi tester PATCH for at opdatere et produkt.
8. Vi tester DELETE for at slette et produkt.
9. Vi samler op på typiske fejl og på, hvordan man læser API-svar.
10. Vi perspektiverer til næste gang, hvor vi flytter requests ind i React-kode.

## Materialer

- README-opgaven i dette repository.
- Thunder Client i VS Code.
- Supabase endpoint og publishable key.
- Computer med adgang til projektet.
- Eksempler på JSON-body til POST og PATCH.
- Oversigt over statuskoder som `200`, `201` og `204`.
